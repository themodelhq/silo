# Deploying PayEnvelope

This repo is set up to deploy as two independent services:

- **Frontend** (`frontend/`) → **Netlify** (static PWA hosting)
- **Backend** (`backend/`) → **Render** (Python web service + managed Postgres)

They work standalone — the frontend runs entirely offline against IndexedDB
with zero configuration. Wiring them together is optional (see the last
section below).

Push this repo to GitHub (or GitLab/Bitbucket) first; both platforms deploy
from a Git repo.

---

## 1. Deploy the frontend to Netlify

`netlify.toml` at the repo root already configures everything: it points
Netlify at the `frontend/` folder, sets cache headers so the service worker
always re-checks for updates, and caches icons aggressively.

1. Go to [app.netlify.com](https://app.netlify.com) → **Add new site** →
   **Import an existing project**.
2. Pick this repository. Netlify auto-detects `netlify.toml` — you shouldn't
   need to touch the build settings (base directory `frontend`, no build
   command, publish directory is that same folder).
3. Click **Deploy site**.
4. Once it's live, you'll get a URL like `https://random-name.netlify.app`.
   Rename it under **Site configuration → Change site name** if you'd like
   something like `payenvelope.netlify.app`.

That's it — the PWA is live, installable, and works offline. No environment
variables are required for the frontend by default.

### Verifying the PWA install works
Open the deployed URL in Chrome/Edge on desktop or Android and you should
see an install prompt (or the install icon in the address bar). On iOS
Safari, use Share → **Add to Home Screen**.

## 2. Deploy the backend to Render

`render.yaml` at the repo root defines both the web service and a managed
Postgres database as a single Blueprint.

1. Go to [dashboard.render.com](https://dashboard.render.com) → **New +** →
   **Blueprint**.
2. Connect the same repository. Render finds `render.yaml` automatically and
   shows you a preview of what it'll create: a `payenvelope-api` web service
   and a `payenvelope-db` Postgres database.
3. Click **Apply**. Render provisions the database first, then builds and
   deploys the API (`pip install -r requirements.txt`, then
   `uvicorn app.main:app --host 0.0.0.0 --port $PORT`).
4. `JWT_SECRET_KEY` is auto-generated by the Blueprint (`generateValue: true`)
   — you don't need to set it yourself.
5. Once deployed, your API is live at `https://payenvelope-api.onrender.com`
   (or whatever name you gave it). Visit `/docs` for interactive Swagger
   docs, and `/health` to confirm it's up.

### Free-tier notes
- The free web service **spins down after 15 minutes of inactivity** — the
  first request after that takes ~30-60s while it wakes back up. Upgrade the
  `plan` field in `render.yaml` to a paid tier to avoid this.
- The free Postgres database **expires after a limited window** (Render has
  changed this policy over time — check current terms at
  [render.com/pricing](https://render.com/pricing) before relying on it for
  anything beyond testing).

## 3. Lock down CORS between them

By default `render.yaml` sets `ALLOWED_ORIGINS=*` so the API works
immediately for testing. Once you know your Netlify URL:

1. In the Render dashboard, open the `payenvelope-api` service →
   **Environment**.
2. Set `ALLOWED_ORIGINS` to your Netlify URL(s), comma-separated if you have
   more than one (e.g. a custom domain plus the `.netlify.app` subdomain):
   ```
   ALLOWED_ORIGINS=https://payenvelope.netlify.app,https://payenvelope.com
   ```
3. Save — Render redeploys automatically with the new value.

This matters because browsers reject credentialed cross-origin requests
against a wildcard `*` origin; `backend/app/main.py` already handles this by
disabling `allow_credentials` while the wildcard is in place, but you'll
want real origins set before relying on this in production.

## 4. (Optional) Connect the frontend to the backend

As shipped, the frontend never calls the backend — it's a fully self-contained
offline demo using IndexedDB, and the backend is a fully working, separately
testable API. If you want the deployed frontend to actually authenticate and
sync through the deployed backend instead:

1. Open `frontend/js/config.js` and set:
   ```js
   const PAYENVELOPE_CONFIG = {
     API_BASE_URL: "https://payenvelope-api.onrender.com",
   };
   ```
2. Update the relevant handlers in `frontend/js/app.js` (`handleLogin`,
   `handleRegister`, and the payslip/envelope/transaction save functions) to
   `fetch()` the corresponding backend endpoints instead of (or alongside)
   writing straight to IndexedDB. The backend's Swagger docs at `/docs` show
   the exact request/response shape for every endpoint.
3. Redeploy the frontend (push to your connected branch — Netlify redeploys
   automatically).

This isn't done for you in this build since it's a real integration task
(token storage, offline-queue-then-sync semantics, error handling) rather
than a config change — `js/config.js` just gives you the one place to start.
